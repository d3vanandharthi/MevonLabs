### Revised Design System

```
Typography:
───────────
  Headings:  "Plus Jakarta Sans" — modern, friendly, readable
  Body:      "Inter" — yes, I said never use Inter before,
             but for YOUR audience readability > uniqueness
  Accent:    "DM Mono" for prices and technical details

  Actually — let me re-think:
  Headings:  "Outfit" — geometric but warm, not cold
  Body:      "Source Sans 3" — clean, professional, very readable
  Mono:      "JetBrains Mono" for prices/badges

Color:
──────
  Background:    #FFFFFF (white — clean, trustworthy, accessible)
  Surface:       #F8F9FA (light gray sections)
  Accent:        #2563EB (confident blue — trust + tech)
  Accent Hover:  #1D4ED8
  Text Primary:  #111827 (near-black)
  Text Muted:    #6B7280
  Success:       #059669 (green for "included" items)
  Border:        #E5E7EB

             Think: Stripe, Linear, every bank website.

  Background:    #FFFBF5 (warm off-white)
  Accent:        #E85D04 (warm orange — energetic, friendly)
  This works better for local vendors / food businesses.

  DECISION: Go with WHITE + ACCENT BLUE for versatility.
            Can add warm tones in imagery.
```

## Technical Stack (Revised)

```
Chain of Thought:
─────────────────
Q: Does your audience care about tech stack?
A: Absolutely not. But it affects speed, cost, and maintenance.

Options:
  1. Pure HTML/CSS/JS — cheapest hosting, fastest, hardest to update
  2. Astro — great but overkill for this
  3. Next.js — way overkill
  4. WordPress — client-familiar, easy to update, but slow
  5. Static site with a CMS — best of both worlds

DECISION:
─────────
  Build with: Plain HTML + CSS + minimal JS
  Host on:    Cloudflare Pages (free) or Netlify (free)
  CMS:        None for now — you update the code directly
  WhatsApp:   wa.me link with pre-filled message
  Forms:      Formspree (free) or Netlify Forms (free)
  Analytics:  Plausible (free tier) or just Google Analytics

WHY:
  - Zero cost to run
  - Blazing fast (static HTML)
  - No WordPress security headaches
  - No database to maintain
  - You can hand-code updates in minutes
  - When you grow, migrate to Astro/Next.js later

  For a 2-person team at 10K INR projects,
  you need ZERO overhead on your own site.
```
